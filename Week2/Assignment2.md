# Fix The Game!

In the demos folder, you will find the rock, paper, scissors game we worked on together in class! Your assignment is to improve the program. While it does technically run, as we dicussed in class, there were many UI/UX flaws, logic flaws and no input validation.

Your task is to fix the program so a user who does not know anything about programming can run the code, and play the game. 

Make the UI as user-friendly as possible, and as interactive as possible. **And** make sure you jusitfy your choices in the design doc. You must also explain what parts of the current code needs improvements and why, in the design doc. 

## Tasks
[ ] Improve UI/UX (Explain your choices in the design doc)  
[ ] Reduce Code Redundancy    
[ ] Add Input Validation   
[ ] Improve game logic   
[ ] One bonus feature to make your game stand out    

**NOTE**: Do not drastically change the game, it must still follow the typical rock, paper, scissor structure. You are only allowed to use topics we have learned in Weeks 1 and 2. 

## Submission
[ ] Github link to the Assignment source file in Brightspace  
[ ] PDF or Text file of the design doc, attached in Brightspace  

**AI IS ALLOWED**, but you must document AI usage in the design doc. Remember to use AI as a senior developer who can guide you, not as a mindless code generator!
